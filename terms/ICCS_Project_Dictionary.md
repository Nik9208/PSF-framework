# 📘 ICCS / PSF — DICTIONARY OF MEANINGS (v0.2)

---

# 1. MAIN IDEA OF THE ENTIRE SYSTEM

## ICCS
It is a way of looking at data like this:
> "A system doesn't just make noise or change — it lives in different states and switches between them."

Simpler:
* there is a data stream
* it is not uniform
* it consists of "behavioral regimes"
* ICCS tries to see these regimes

---

## PSF
It is simply the "box" where everything is stored and works.

Simpler:
> the method itself (ICCS) + the folder/project where it is implemented

---

# 2. BASIC CONCEPTS (THE MOST IMPORTANT MEANING)

## State
It is:
> "what the system looks like right now"

Example:
* calm phase
* noise
* growth
* fall

---

## Regime
It is:
> "a recurring state in which the system lingers"

Simpler:
* not a random moment
* but a character of behavior

Example:
* the market is "growing"
* the market is "falling"
* the system is "jumping"

---

## Transition
It is:
> the moment when the system stops being one thing and becomes another

Important:
* this is not just a change
* this is a change in the "regime of life"

---

## Memory
It is:
> how much the past affects the present

Simpler:
* if there is memory → the system "remembers"
* if there is none → everything is random

---

## Dependency
It is:
> whether parts of the system are connected to each other

Example:
* one variable affects another
* or everything is independent

---

# 3. "PICTURE IN THE HEAD" (MAIN POINT)

## Geometry
This is NOT math in the strict sense.

It means:
> "what the regimes look like if you imagine them as a map"

Simpler:
* regimes = points/areas
* transitions = lines between them

👉 this is simply a "map of the system's behavior"

---

# 4. OPERATORS (MEANING OF MECHANISMS)

## SCO
It is:
> a way to "gather similar pieces of data into groups"

Simpler:
* we take chaos
* glue together what is similar
* get regimes

---

## RSO
It is:
> a check of "how well a regime holds up"

Simpler:
* stable regime = lives long
* unstable = quickly falls apart

---

## TOO
It is:
> a check of "whether there is a direction of time"

Simpler:
* the system can behave differently forwards and backwards
* if there is a difference → the process is "irreversible"

---

# 5. TRUTH VERIFICATION

## BVP / Falsification
It is:
> a check of "whether we invented structure where there is none"

Simpler:
* we take noise
* shuffle the data
* check: "are the regimes still there or have they disappeared?"

👉 if they disappeared → it was an illusion
👉 if they remained → it's possibly a real structure

---

# 6. MODULES

## Module C (the most important meaning)
It is:
> an attempt to draw a "map of the system's behavior"

Simpler:
* not just finding regimes
* but understanding how they are located and connected

---

## Epistemic Compiler
It is:
> a "translator" from raw data into understandable features

Simpler:
* raw stream → structured description

---

# 7. VERY IMPORTANT TERMS (THE MOST HONEST DICTIONARY)

## Feature
It is simply:
> a characteristic of the data

---

## Cluster
It is:
> a group of similar points

---

## Phase Space
It is:
> an imaginary space of all possible states

---

## Null Model
It is:
> "what if everything were random"

Used as a control baseline.

---

## Surrogate Data
It is:
> intentionally corrupted data for testing purposes

---

# 8. SUPER-SHORT FOR THE ENTIRE SYSTEM

ICCS can be explained in one sentence:
> the system looks for recurring "behavioral regimes" in data and checks whether they are real or random.

---

# 📌 IMPORTANT SUMMARY

Your system is not "complex mathematics".

It is:
> a way to slice a stream of data into "states" and understand how they replace each other.

# 📘 ICCS / PSF — СЛОВАРЬ СМЫСЛОВ (v0.2)

---

# 1. ГЛАВНАЯ ИДЕЯ ВСЕЙ СИСТЕМЫ

## ICCS
Это способ смотреть на данные так:
> “Система не просто шумит или меняется — она живёт в разных состояниях и переключается между ними.”

Проще:
* есть поток данных
* он не одинаковый
* он состоит из “режимов поведения”
* ICCS пытается эти режимы увидеть

---

## PSF
Это просто “коробка”, где всё хранится и работает.

Проще:
> сам метод (ICCS) + папка/проект, где он реализован

---

# 2. ОСНОВНЫЕ ПОНЯТИЯ (САМЫЙ ВАЖНЫЙ СМЫСЛ)

## State (состояние)
Это:
> “как система выглядит прямо сейчас”

Пример:
* спокойная фаза
* шум
* рост
* падение

---

## Regime (режим)
Это:
> “повторяющееся состояние, в котором система задерживается”

Проще:
* не случайный момент
* а характер поведения

Пример:
* рынок “растёт”
* рынок “падает”
* система “скачет”

---

## Transition (переход)
Это:
> момент, когда система перестаёт быть одной и становится другой

Важно:
* это не просто изменение
* это смена “режима жизни”

---

## Memory (память)
Это:
> насколько прошлое влияет на настоящее

Проще:
* если есть память → система “помнит”
* если нет → всё случайно

---

## Dependency (зависимость)
Это:
> связаны ли части системы между собой

Пример:
* одна переменная влияет на другую
* или всё независимо

---

# 3. “КАРТИНКА В ГОЛОВЕ” (ГЛАВНОЕ)

## Geometry (геометрия)
Это НЕ математика в строгом смысле.

Это означает:
> “как выглядят режимы, если представить их как карту”

Проще:
* режимы = точки/области
* переходы = линии между ними

👉 это просто “карта поведения системы”

---

# 4. ОПЕРАТОРЫ (СМЫСЛ МЕХАНИЗМОВ)

## SCO
Это:
> способ “собрать похожие куски данных в группы”

Проще:
* берём хаос
* склеиваем похожее
* получаем режимы

---

## RSO
Это:
> проверка “насколько режим держится”

Проще:
* устойчивый режим = долго живёт
* неустойчивый = быстро разваливается

---

## TOO
Это:
> проверка “есть ли направление времени”

Проще:
* система может вести себя по-разному вперёд и назад
* если разница есть → процесс “необратимый”

---

# 5. ПРОВЕРКА ПРАВДИВОСТИ

## BVP / Falsification
Это:
> проверка “не придумали ли мы структуру там, где её нет”

Проще:
* берём шум
* перемешиваем данные
* проверяем: “режимы всё ещё есть или исчезли?”

👉 если исчезли → это была иллюзия
👉 если остались → возможно реальная структура

---

# 6. МОДУЛИ

## Module C (самый важный смысл)
Это:
> попытка нарисовать “карту поведения системы”

Проще:
* не просто найти режимы
* а понять, как они расположены и связаны

---

## Epistemic Compiler
Это:
> “переводчик” из сырых данных в понятные признаки

Проще:
* сырой поток → структурированное описание

---

# 7. ВАЖНЫЕ ВАЖНЫЕ ТЕРМИНЫ (САМЫЙ ЧЕСТНЫЙ СЛОВАРЬ)

## Feature
Это просто:
> характеристика данных

---

## Cluster
Это:
> группа похожих точек

---

## Phase Space
Это:
> воображаемое пространство всех возможных состояний

---

## Null Model
Это:
> “если бы всё было случайно”

Используется как контроль

---

## Surrogate Data
Это:
> специально испорченные данные для проверки

---

# 8. СУПЕР-КОРОТКО ВСЯ СИСТЕМА

ICCS можно объяснить одной фразой:
> система ищет повторяющиеся “режимы поведения” в данных и проверяет, настоящие они или случайные.

---

# 📌 ВАЖНЫЙ ИТОГ

Твоя система — это не “сложная математика”.

Это:
> способ разрезать поток данных на “состояния” и понять, как они сменяют друг друга.
